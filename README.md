# ALB — Application Load Balancer

High-performance load balancer with hot-swappable strategies and DPDK packet processing.

## Quick Start

```bash
# Prerequisites (Ubuntu 22.04)
sudo apt-get install -y dpdk-dev libnuma-dev cmake git build-essential
sudo curl -fSL https://github.com/bazelbuild/bazelisk/releases/latest/download/bazelisk-linux-amd64 \
  -o /usr/local/bin/bazel && sudo chmod +x /usr/local/bin/bazel

# Build and install llama.cpp from source
sudo ./third_party/install_llama_cpp.sh

# Download Llama 3.1 8B Instruct GGUF into ./models/ (not tracked by git)
./scripts/download_llama_model.sh

# Build & test
bazel build //...
bazel test //...
```

## Traffic Generator

```bash
bazel-bin/packages/traffic-generator/traffic-generator --vdev=net_null0 -l 0,1,2
```

Requires 3+ lcores: main (stats), workers (TX), last worker (RX drain).

## Tests

| Test | Description |
|------|-------------|
| `config_test` | Parses YAML configs, verifies IP/port/mac/weight parsing |
| `strategy_test` | Loads strategy via dlopen, verifies round-robin routing |
| `traffic-generator_build_test` | Verifies DPDK linkage |
| `eal_init_test` | DPDK EAL init with virtual device |
| `tx_test` | Verifies nonzero TX throughput |

DPDK tests use `--no-huge --no-pci --vdev=net_null0` — no real NICs or hugepages needed.

## Code Generation

Strategy `.so` files are generated by a self-hosted [llama.cpp](https://github.com/ggml-org/llama.cpp) build using **[Llama 3.1 8B Instruct](https://huggingface.co/meta-llama/Llama-3.1-8B-Instruct)** weights in **GGUF** form. The generated code is compiled and hot-swapped into the balancer at runtime via inotify + dlopen.

Weights are **not** committed. After installing llama.cpp (`./third_party/install_llama_cpp.sh`), download the GGUF into `./models/` (same base model as the Meta card above; file comes from the public [bartowski GGUF repo](https://huggingface.co/bartowski/Meta-Llama-3.1-8B-Instruct-GGUF) and you must follow Meta’s license):

```bash
./scripts/download_llama_model.sh
./scripts/verify_llama_model.sh
```

Default path: `models/Meta-Llama-3.1-8B-Instruct-Q4_K_M.gguf`. Override with `ALB_GGUF_PATH` for tools that read the model path.
